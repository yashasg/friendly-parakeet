PATTERN LEARNING — KEATON
================================================================================

Title: Integration Tests Must Exercise the Production Tick Path

Context:
  When a refactor introduces a new event-emit + consume system pair, the 
  production-loop wiring is the most-easily-missed integration point.

Lesson:
  1. Write a PRODUCTION-TICK integration test FIRST (not unit tests first)
  2. Unit tests that self-wire systems can mask a missing production call
  3. The litmus test for wiring: if the test passes when the production call 
     is deleted, the test measures the wrong thing
  4. Demand at least one test that exercises the actual production tick path
     (or as close as linking constraints allow)

Application:
  - Integration test "lane push consumed in production tick order" directly 
    exercises collision → response → miss_detection in sequence
  - All six existing unit tests annotated to clarify they are unit-scoped 
    (self-wired), while integration test is production-path verification

Evidence:
  - Test would FAIL before the wiring fix
  - Test now PASSES, proving the production path is correct
  - Wiring is the first integration test, not the last

Title: Tag-Replacement Pattern Ratified (Third Successful Application)

Context:
  Replacing inline enum-kind branches with ECS tags is now a proven pattern 
  with three successful applications across consecutive rounds.

Pattern Evolution:
  Round 5: NonScorableTag replaces (kind == LanePushLeft || kind == LanePushRight)
  Round 6: LanePushDelta replaces (kind == LanePushLeft/Right) — captures direction
  Round 7: BarObstacleTag replaces (kind == LowBar || kind == HighBar)

Recipe (Standardized):
  1. Identify a kind-based branch in a system (kind == X || kind == Y)
  2. Create an empty tag struct in app/components/ with doc comment
  3. Emplace the tag at spawn-time (obstacle_entity.cpp) on matching arms
  4. Replace the kind-read with reg.any_of<TagName>(entity) in the system
  5. Write synthetic-kind + tag tests proving kind-independence + else-branch
  6. Verify: grep -n 'ObstacleKind::' <system>.cpp returns zero matches
