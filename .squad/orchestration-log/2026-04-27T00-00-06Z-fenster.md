## Work Summary

Fenster identified and fixed the beat-grid anchor ambiguity in the level-design pipeline. The scheduler's uniform-period formula (`arrival_time = offset + beat_index * beat_period`) had been blindly anchoring to `beats[0]` from aubio detection—vulnerable to false early detections that would shift the entire beat grid. Fenster defined offset explicitly as "audio time of beat_index=0" and retooled `level_designer.py` to compute it from the minimum authored beat index across all difficulties.

### Changes

- **`app/components/beat_map.h`** — Added doc comment to `offset` field:
  ```
  /// Audio time (seconds from track start) of beat_index = 0 in the uniform beat grid.
  /// Computed as: offset = beats[anchor_idx] - anchor_idx * beat_period
  /// where anchor_idx = min(all authored beat_index values).
  ```

- **`app/systems/beat_scheduler_system.cpp`** — Added comment at arrival_time site explaining offset semantics and anchor invariant.

- **`tools/level_designer.py`** — `build_beatmap()` refactored:
  - Previously: `offset = round(beats[0], 3)` (blind first detection)
  - Now: `anchor_idx = min(beat_index over all authored obstacles)`; `offset = round(beats[anchor_idx] - anchor_idx * beat_period, 3)`
  - Robust against false early detections; future-proofs content pipeline

- **`tools/validate_beatmap_offset.py`** — **New** authoring-time validation tool:
  - Checks all shipped beatmaps have correctly anchored offsets (tolerance: 2ms)
  - Command: `python3 tools/validate_beatmap_offset.py [--all] [file1.json ...]`
  - Exit 0 on pass; stderr diagnostic on fail
  - Integrated into CI gates

- **`content/beatmaps/*.json`** — All 3 shipped beatmaps regenerated:
  - `stomper.json`: offset 2.270 → 2.269s (1ms delta; imperceptible)
  - `drama.json`, `mental.json`: offset unchanged
  - All collisions remain within ±1ms of analyzed onsets
