# Orchestration Log — Saul (Design Target)

**Date:** 2026-04-27  
**Role:** Game Designer  
**Issue:** #135 — Difficulty Ramp: Easy Variety + Medium LanePush  

## Outcome

**Status:** ✅ APPROVED

Saul defined the design contract for #135:

- **Easy:** Controlled variety **inside shape_gate space only**. Allowed kinds strictly `{shape_gate}`. No lane_push. No bars. Constraints: ≥20% non-center lane distribution, no >6 consecutive same-lane, no >12 consecutive same-shape, no single gap >50%, section density variation.

- **Medium:** LanePush becomes a *taught mechanic*. Shape-only intro (no lane_push before max(30s, first chorus)). LanePush soft cap 20% share. Min gap 3 beats between consecutive pushes; 4-beat telegraph window around first 3 occurrences. Push direction safety: no wall-pushes.

- **Hard:** Unchanged from #125; bars remain hard-only.

## Deliverables

- Design target documented in `.squad/decisions/inbox/saul-difficulty-ramp-135.md`
- Constraints operationalized as Python heuristics in `tools/level_designer.py` (implementation by Rabin)
- Test player validation criteria specified (easy ≥90% completion, medium first-LanePush ≥80% survival)

## Key Decisions

1. Easy gains *variety*, not *complexity*. Variety is rhythm-driven (section density, gap distribution, lane distribution) and shape-driven (3-shape rotation), not mechanic-driven.
2. Medium LanePush is gated: shape-only intro, capped share, safe directions, readable spacing.
3. Hard bars already settled in #125; no changes.
