## Key decisions

- iOS is the only customer-facing TestFlight/App Store v1 platform.
- The iOS persistence requirement is app-sandbox JSON through the shared C++ filesystem abstraction; no iCloud, Core Data, or NSUserDefaults guarantee is claimed.
- Desktop persistence is CI/dev only and follows existing filesystem directories.
- WASM persistence is CI/dev only and uses current-directory Emscripten VFS; durable browser storage is future work.
- Current persisted data is high scores/settings only.
- FTUE run count is required before v1 but is not implemented yet.
